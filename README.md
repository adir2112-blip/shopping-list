# רשימת קניות 🛒

אפליקציית רשימת קניות משפחתית. נתונים משותפים בזמן אמת בין כל המכשירים (דרך Supabase), עם תמיכה בכמה משפחות נפרדות, מאגר מוצרים כללי, ורשימת קניות פעילה.

## איך זה עובד - העקרון המרכזי

יש **מאגר כללי** של כל המוצרים שהוגדרו אי פעם (בלשונית "עריכת רשימה"), ויש **רשימת קניות פעילה** (בלשונית "רשימת קניות") שמכילה רק את מה שנבחר לקניה הקרובה.

בעריכה, מסמנים ✓ ליד מוצרים מהמאגר ולוחצים "צור רשימת קניות" — הם עוברים לרשימה הפעילה. כשמוסיפים מוצר חדש תוך כדי קניה בסופר (משהו שלא היה ברשימה), הוא מתווסף גם למאגר הכללי לצמיתות, כך שבפעם הבאה אפשר לסמן אותו שוב.

## מבנה הפרויקט

```
login.html      → מסך בחירת משפחה + קוד כניסה (פותחים את זה ראשון)
index.html      → רשימת קניות (פותחים את זה בסופר)
edit.html       → עריכת רשימת קניות (מאגר כללי, סימון, גרירה, הוספה/מחיקה, תמונות)
history.html    → היסטוריית קניות שהושלמו
settings.html   → הגדרות (גלוי רק למשפחת אדיר ויקירה) - ניהול משפחות ומספרי טלפון להתראות
seed.html       → טעינה חד-פעמית של רשימת המוצרים הראשונית (למשפחת אדיר ויקירה)
manifest.json   → הגדרות PWA (מאפשר "הוספה למסך הבית" כאפליקציה)
schema.sql      → מבנה הטבלאות להתקנה חדשה מאפס
migration-workspaces.sql            → מיגרציה: workspace_id בטבלאות הקיימות
migration-quantity-and-active-list.sql → מיגרציה: כמות + הפרדת מאגר/רשימה פעילה
migration-product-images.sql        → מיגרציה: תמיכה בתמונות מוצר
migration-workspaces-db.sql         → מיגרציה: טבלת משפחות + טבלת הגדרות (להפעלת לשונית הגדרות)
STORAGE-SETUP.md → הוראות הקמת אחסון תמונות ב-Supabase (קוראים לפני המיגרציה)
assets/         → CSS, JS, אייקוני PWA, ונתוני ה-seed
```

## שלב 1: הקמת אחסון תמונות (Storage)

קוראים ועוקבים אחרי `STORAGE-SETUP.md` — יוצרים bucket בשם `product-images` ומגדירים הרשאות גישה. **חשוב לעשות את זה לפני המיגרציה הבאה.**

## שלב 2: הקמת מסד הנתונים ב-Supabase

**אם זו התקנה חדשה לגמרי (אין עדיין טבלאות):**
1. נכנסים לפרויקט ב-[supabase.com](https://supabase.com)
2. **SQL Editor** → **New query**
3. מדביקים את כל התוכן של `schema.sql` ולוחצים **Run**

**אם יש כבר טבלאות עם דאטה קיים (כמו אצלכם):**
מריצים את ארבע המיגרציות הבאות, אחת אחרי השניה, ב-**SQL Editor**:
1. `migration-workspaces.sql`
2. `migration-quantity-and-active-list.sql`
3. `migration-product-images.sql`
4. `migration-workspaces-db.sql`

שום דבר לא נמחק בתהליך.

## שלב 3: חיבור האפליקציה לפרויקט שלכם

ב-Supabase: **Settings → API**, מעתיקים **Project URL** ו-**anon public key**, ומדביקים ב-`assets/supabase-client.js`:

```js
const SUPABASE_URL = "https://xxxxx.supabase.co";
const SUPABASE_ANON_KEY = "eyJ...";
```

(זה כבר מוגדר בקבצים שקיבלתם, עם הפרטים של הפרויקט שלכם.)

## שלב 4: ניהול המשפחות וקודי הכניסה

המשפחות מנוהלות עכשיו ב-Database (לא בקוד) ונטענות אוטומטית בכל הדפים. שלוש המשפחות הקיימות נוצרות אוטומטית ע"י `migration-workspaces-db.sql` עם הקודים שכבר היו מוגדרים (אדיר ויקירה, גל, חיה).

**להוספת משפחה חדשה או איפוס קוד כניסה** — נכנסים למשפחת אדיר ויקירה, ולוחצים על לשונית **"הגדרות"** (גלויה רק להם). משם אפשר ליצור משפחות חדשות עם שם, אימוג'י וקוד, ולאפס קוד לכל משפחה קיימת — בלי לערוך קוד או להעלות קבצים מחדש.

זו הגנה בסיסית למשפחה פרטית, לא אבטחה קריפטוגרפית. כל משפחה רואה ועורכת רק את הדאטה שלה.

## שלב 5: טעינה ראשונית (רק למשפחת אדיר ויקירה)

אחרי ההעלאה ל-GitHub Pages (שלב הבא), פותחים את `seed.html` מהכתובת המקוונת (לא מהמחשב) ולוחצים על הכפתור. זה טוען את 11 המחלקות וכל המוצרים, ומסמן אותם כחלק מהרשימה הפעילה. **אם כבר הרצתם את זה בעבר, אין צורך לחזור על השלב הזה.**

## שלב 6: העלאה ל-GitHub Pages

### העלאה ראשונית (ריפו חדש)
1. יוצרים repository חדש ב-GitHub (Public)
2. **Add file → Upload files**, גוררים את **כל התוכן** של התיקייה (כל קבצי ה-html/sql/md, ותיקיית `assets` כיחידה שלמה) לתוך אזור ההעלאה
3. **Commit changes**
4. **Settings → Pages**: Source = `Deploy from a branch`, Branch = `main`, תיקיה = `/ (root)`, **Save**
5. אחרי דקה-שתיים: `https://USERNAME.github.io/REPO_NAME/login.html`

### עדכון ריפו קיים (מומלץ: מחיקה והעלאה מחדש, כדי למנוע קבצים ישנים שנשארים)
כדי להבטיח שאין קבצים ישנים/חלקיים שנשארים מעדכון קודם:
1. בריפו הקיים, פותחים כל קובץ בנפרד ובודקים את שלושת הנקודות (⋮) → **Delete file**, וחוזרים על זה לכולם — **או** הדרך המהירה: גוררים את **כל הקבצים החדשים** (כולל תיקיית `assets`) לתוך **Add file → Upload files** ומ-commit. GitHub יחליף קבצים עם אותו שם אוטומטית, ויוסיף קבצים חדשים. קבצים שלא נכללים בהעלאה החדשה אך כן קיימים בריפו (כמו קובץ ישן שלא רלוונטי יותר) יישארו — אבל זה לא בעיה כל עוד כל הקבצים החדשים עלו.
2. חשוב: ודאו שתיקיית `assets` עולה **בשלמותה** עם כל 7 הקבצים שבתוכה (`style.css`, `shop.js`, `edit.js`, `history.js`, `login.js`, `supabase-client.js`, `workspaces.js`, `seed-data.browser.js`) — זו הטעות הנפוצה ביותר.
3. אחרי ה-commit, פותחים את האתר עם **רענון מלא** (לא רק F5 — בנייד: לסגור את הלשונית ולפתוח מחדש; במחשב: Ctrl+Shift+R) כדי לנקות cache ישן.

## שימוש יומיומי

**כניסה** (`login.html`) — כל בן משפחה בוחר את הכרטיס שלו ומקליד קוד 4 ספרות. הבחירה נשמרת במכשיר. לחיצה על תג השם בראש כל דף מתנתקת ומחזירה למסך הבחירה.

**עריכת רשימת קניות** (`edit.html`) — כאן מנהלים את **כל** המוצרים שאי פעם הוגדרו, מחולקים למחלקות. לכל מוצר: checkbox (האם לכלול ברשימה הקרובה), תמונה קטנה (לחיצה עליה פותחת מודל להוספה/החלפה/הסרה של תמונה — מצלמה או גלריה), שם, כמות, מחיר, וכפתור מחיקה. אפשר לגרור מוצרים בין מחלקות ולשנות את סדר המחלקות. שדה חיפוש בראש העמוד מסנן מוצרים בזמן הקלדה. בתחתית מוצג סכום כל המוצרים המסומנים ("צפי לקניה הבאה") לפני שמתחייבים. לאחר סימון כל מה שרוצים לקניה הקרובה, לוחצים **"צור רשימת קניות"** — המוצרים המסומנים עוברים לרשימה הפעילה (מצטבר, לא מוחק קודמים), והסימון שלהם במאגר מתאפס.

**רשימת קניות** (`index.html`) — פותחים בסופר. מציג רק את מה שהועבר מהעריכה. למוצרים עם תמונה מוצג thumbnail קטן; לחיצה עליו פותחת את התמונה במסך מלא. לחיצה על שאר השורה עוברת בין שלושה מצבים: ממתין → נקנה (✓, ירוק) → חסר במלאי (!, כתום) → חזרה לממתין. מוצרים "חסרי מלאי" מופיעים גם במחלקה שלהם וגם בבלוק נפרד בסוף הרשימה. הסכום מחושב כ-מחיר × כמות. בתחתית: **צפי רשימה** (סכום קבוע) ו**סכום קניה נוכחית** (עולה עם כל מוצר שנקנה). בכל מחלקה אפשר להוסיף מוצר שלא היה ברשימה — הוא מצטרף גם לרשימה הפעילה וגם למאגר הכללי בעריכה. בסיום, **"סיימתי קניה"** שומר את הרשומה בהיסטוריה; מוצרים שנקנו חוזרים למאגר הכללי (לא מסומנים), ומוצרים "חסרי מלאי" נשארים ברשימה הפעילה כתזכורת.

**היסטוריה** (`history.html`) — כל הקניות שהושלמו, עם סכומים ופירוט מוצרים (כולל כמות).

**הגדרות** (`settings.html`, גלוי רק למשפחת אדיר ויקירה) — יצירת משפחות חדשות (שם, אימוג'י, קוד כניסה), איפוס קוד למשפחה קיימת, וניהול מספרי טלפון לכל משפחה לקבלת התראת וואטסאפ על חוסרי מלאי באותה משפחה בלבד.

**התראת וואטסאפ על חוסרי מלאי** — אם הוגדרו מספרי טלפון למשפחה מסוימת בהגדרות, בכל פעם שאותה משפחה מסיימת קניה עם מוצרים שסומנו "חסר במלאי", נשלחת אוטומטית הודעת וואטסאפ למספרים שהוגדרו לה (ולא למשפחות אחרות), עם רשימת המוצרים החסרים. השליחה מתבצעת דרך בוט קיים (whatsapp-web.js) ולא חוסמת את שמירת הקניה — אם השליחה נכשלת, הקניה עדיין נשמרת כרגיל.

**התקנה כאפליקציה** — בכניסה הראשונה לאתר, יופיע באנר קטן בראש המסך שמציע "התקינו את האפליקציה". באנדרואיד/כרום, לחיצה על "התקנה" מתקינה את האפליקציה בפועל למסך הבית; באייפון (Safari), הבאנר מסביר איך לעשות זאת ידנית (שיתוף ⬆️ → הוסף למסך הבית), כי אפל לא מאפשרת התקנה אוטומטית מהדפדפן. אם המשתמש לוחץ "התקנה"/"הבנתי", הבאנר לא יופיע יותר; אם הוא סוגר אותו (✕) בלי לאשר, הוא יופיע שוב בכניסה הבאה.

## הערות טכניות

- כל שינוי נשמר ישירות ל-Supabase ומסונכרן בזמן אמת לכל מכשיר פתוח **באותה משפחה**
- כל משפחה (workspace) רואה ועורכת רק את הדאטה שלה
- מצב הקיפול/פתיחה של מחלקות הוא מקומי למכשיר (לא משותף)
- האפליקציה היא קובץ סטטי טהור — אין צורך בשרת, build step, או npm install
